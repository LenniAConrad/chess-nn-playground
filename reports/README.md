# Reports

This directory stores generated summaries across runs. It is separate from `results/`, which stores per-run artifacts and checkpoints.

## Main Outputs

- `leaderboards/global_leaderboard.md`: human-readable leaderboard rebuilt by `scripts/compare_results.py`.
- `leaderboards/global_seed_summary.md`: seed-level aggregation when repeated seeds exist.
- `latest/latest_report.md` and `latest/latest_report.html`: latest run report copies.
- `prompts/chatgpt_pro_run_prompt.md`: generated handoff prompt built from current results.
- `training/`: aggregate plots from `scripts/reports/plot_training_results.py`.
- `experiment_logs/`: subprocess logs from suite runs.
- `experiment_suites/`: suite-level run summaries.
- `paper_ready_all/`: resumable all-benchmarks/all-ideas plans, logs, generated configs, event timelines, and state.
- `archive/`: historical one-off reports and old logs kept for provenance.

For paper-ready batches, open `paper_ready_all/status.md` first. It is regenerated by `scripts/run_paper_ready_all.py` and links the plan, state file, timestamped event log, readable timeline, task logs, generated configs, ETA, leaderboard, seed summary, training dashboards, and `paper_ready_all/paper_report.pdf`.

## Rebuild Commands

```bash
PYTHONDONTWRITEBYTECODE=1 python scripts/compare_results.py --results-dir results

PYTHONDONTWRITEBYTECODE=1 python scripts/reports/plot_training_results.py \
  --results-dir results \
  --output-dir reports/training

PYTHONDONTWRITEBYTECODE=1 python scripts/reports/build_paper_report.py \
  --results-dir results \
  --state-path reports/paper_ready_all/state.json \
  --output reports/paper_ready_all/paper_report.pdf
```

Generated reports are evidence only when the referenced run directories pass artifact validation.

The rebuild commands scan nested run directories, so `--results-dir results` includes paper-ready task outputs under `results/paper_ready_all/`.
