# Codex Handoff Packet: Support-Polar Zonotope Certificate Network

## 1. File Metadata

**Idea name:** Support-Polar Zonotope Certificate Network, abbreviated **SPZC-Net**

**Task:** binary chess puzzle classification from current-board tensor only

**Model key:** `spzc_zonotope`

**Primary implementation file:** `models/support_polar_zonotope.py`

**Expected model class:** `SupportPolarZonotopeClassifier(nn.Module)`

**Forward signature:** `forward(x: torch.Tensor) -> torch.Tensor`

**Input shape:** `(batch, C, 8, 8)`

**Output shape:** `(batch, 1)` logits

**Target remap:** fine labels `0` and `1` map to binary `0`; fine label `2` maps to binary `1`

**Core convex object:** a board-dependent latent **zonotope** tested against a learned **polar margin body** through support-function containment violations.

**Primary claim:** label-2 puzzles can be detected by whether the board's latent square-pair interaction zonotope protrudes beyond a learned symmetric convex body representing non-label-2 positions.

## 2. Executive Selection

Build a classifier whose central computation is not average pooling, attention pooling, or a plain convolutional head. SPZC-Net maps the current board into a latent convex body:

\[
Z_x = c_x + \sum_{e \in E} [-g_e(x), g_e(x)].
\]

Here \(Z_x\) is a zonotope generated by learned square-pair interaction vectors. The model also learns a symmetric polar body:

\[
Q = P^\circ,\qquad P=\operatorname{conv}\{\pm u_k/\beta_k\}_{k=1}^K.
\]

It classifies the board by the largest halfspace containment violation:

\[
r(x)=\max_{k,\sigma\in\{+1,-1\}}\left(h_{Z_x}(\sigma u_k)-\beta_k\right).
\]

The logit is a calibrated monotone function of \(r(x)\). A positive prediction means that, in at least one learned geometric direction, the board's latent interaction body extends outside the learned quiet-body boundary.

The architecture also exposes a concrete certificate: the violating direction \(u_k\), the sign \(\sigma\), the violation value, and the square-pair generators contributing most to \(|u_k^\top g_e(x)|\).

## 4. Convex-Geometric Research Background

A zonotope is a convex polytope expressible as a Minkowski sum of line segments. This makes it a natural latent object for a chess board: many local or pairwise relations can combine into a single global convex body. Standard references for zonotopes and support functions include Ziegler, *Lectures on Polytopes*; Schneider, *Convex Bodies: The Brunn-Minkowski Theory*; and Rockafellar, *Convex Analysis*.

Support functions are the key computational trick. For a compact convex set \(A\),

\[
h_A(u)=\sup_{a\in A}\langle u,a\rangle.
\]

They satisfy:

\[
h_{A+B}(u)=h_A(u)+h_B(u),
\]

and containment can be checked by support-function inequalities:

\[
A\subseteq B \iff h_A(u)\le h_B(u)\quad\text{for all }u.
\]

For the proposed zonotope, this gives a closed-form neural computation instead of an inner optimization problem.

The separating-hyperplane theorem gives the certificate logic. If a convex set protrudes beyond a convex body in a direction \(u_k\), then that direction is a separating or margin-violating normal. SPZC-Net turns this into an inspectable neural output.

Related neural references include Amos, Xu, and Kolter, "Input Convex Neural Networks," ICML 2017, and Agrawal et al., "Differentiable Convex Optimization Layers," NeurIPS 2019. Those works motivate neural-convex modeling, but SPZC-Net deliberately avoids a solver-backed layer. Its convex computation is closed form.

Set-style board encoders are also relevant. Zaheer et al., "Deep Sets," NeurIPS 2017, gives a reference point for set functions, but the proposed aggregation is not generic set pooling. It is support-function evaluation of a named convex body.

## 5. Candidate Search Trace

The selected direction was chosen by filtering for four properties.

First, the convex object must be explicit. The board becomes \(Z_x\), a real zonotope, not just an unnamed latent vector.

Second, the classifier must have a geometric test. The condition \(Z_x\subseteq Q\) is checked through finite support-function inequalities.

Third, the model must produce an inspectable certificate. A positive or false-positive example can be traced to \((u_k,\sigma,\beta_k)\) and to the square-pair generators with the largest projected contributions.

Fourth, the implementation must be lightweight enough for Codex to build and test quickly. The core operation is matrix projection plus absolute-value summation. No external solver is required.

Alternative convex candidates considered were learned ellipsoids, convex hulls of token embeddings, and projection residuals onto a learned polytope. The zonotope-polar design was selected because it has the cleanest closed-form support function while still representing many pair interactions.

## 6. Rejected Common Approaches

Rejected for this packet:

- **Plain CNN or ResNet head:** strong baseline, but the research object becomes better feature extraction rather than convex geometry.
- **Attention-only square interaction model:** useful baseline, but attention scores are not a support function, polar body, projection residual, or separating certificate.
- **Engine-distilled model:** invalid because it violates the leakage-safe contract.
- **Move-sequence model:** invalid because the target contract is current-board-only.
- **Generic solver-backed convex layer:** mathematically valid but slower, harder to debug, and less Codex-ready than closed-form support evaluation.
- **Metadata-driven classifier:** invalid because non-board fields can create label leakage.
- **Renamed pooling layer:** invalid because the architecture must introduce a concrete convex object, not a standard reducer with geometric vocabulary attached.

## 7. Mathematical Thesis

Fine label `2` positions should be detectable as **latent convex protrusions**. The hypothesis is that a tactically important position is not merely high in a smooth scalar feature; rather, a small number of square-pair relations create extreme support in one or more learned directions.

Fine labels `0` and `1` should more often produce interaction bodies contained in, or close to, the learned polar margin body:

\[
Q = \{z\in\mathbb{R}^{d_z}: |\langle u_k,z\rangle|\le \beta_k,\; k=1,\dots,K\}.
\]

The model tests:

\[
Z_x \subseteq Q.
\]

If

\[
\max_{k,\sigma}\left(h_{Z_x}(\sigma u_k)-\beta_k\right)>0,
\]

then there exists a learned facet direction where the board's interaction body exceeds the quiet-body boundary. That excess is the classification signal.

This thesis is falsifiable. If label `2` is not characterized by sparse extreme interaction geometry, then the support-polar residual should fail to beat same-budget CNN and token baselines.

## 8. Formal Convex Object

Let the board encoder produce square tokens:

\[
T_x = \{t_i(x)\}_{i=1}^{64},\qquad t_i\in\mathbb{R}^{d_t}.
\]

For each ordered square pair \(e=(i,j)\), \(i\ne j\), construct a generator:

\[
g_{ij}(x)=a_{ij}(x)\,\phi(t_i,t_j,\rho_{ij})\in\mathbb{R}^{d_z},
\]

where:

- \(\rho_{ij}\) is a fixed relative-square encoding;
- \(\phi\) is a learned generator MLP;
- \(a_{ij}(x)\in[0,1]\) is a learned gate.

Define the board zonotope:

\[
Z_x
=
\left\{
c_x+\sum_{i\ne j}\alpha_{ij}g_{ij}(x)
:
\alpha_{ij}\in[-1,1]
\right\}.
\]

Its support function is closed form:

\[
h_{Z_x}(u)
=
\langle u,c_x\rangle
+
\sum_{i\ne j}\left|\langle u,g_{ij}(x)\rangle\right|.
\]

Learn \(K\) normalized directions \(u_k\in\mathbb{R}^{d_z}\) and positive thresholds \(\beta_k>0\). Define:

\[
P=\operatorname{conv}\{\pm u_k/\beta_k\}_{k=1}^K,
\]

\[
Q=P^\circ
=
\{z: |\langle u_k,z\rangle|\le \beta_k\;\forall k\}.
\]

Containment against this finite polar body is tested by:

\[
v_{k,+}=h_{Z_x}(u_k)-\beta_k,
\]

\[
v_{k,-}=h_{Z_x}(-u_k)-\beta_k.
\]

The residual is:

\[
r(x)=\max_k\max(v_{k,+},v_{k,-}).
\]

The binary logit is:

\[
\ell(x)=s\,r(x)+b,\qquad s=\operatorname{softplus}(s_0).
\]

A stronger optional head may consume all violations:

```python
logit = monotone_mlp(torch.cat([v_plus, v_minus], dim=-1))
```

If this optional head is used, constrain its linear weights to be nonnegative so larger violations cannot decrease the positive logit. The minimal experiment should start with the scalar residual head.

## 9. Architecture And Tensor Shapes

Recommended defaults:

```text
B = batch size
C = input channels
d_t = 64 square-token width
d_z = 32 zonotope latent dimension
K = 32 learned polar directions
N = 64 squares
E = N * (N - 1) = 4032 ordered square pairs
```

Tensor path:

```text
x                           [B, C, 8, 8]
BoardStem(x)                [B, d_t, 8, 8]
flatten squares             [B, 64, d_t]
pair expansion              [B, 64, 64, 2*d_t + rel_dim]
pair mask i != j            [1, 64, 64, 1]
generator MLP               [B, 64, 64, d_z]
gate MLP                    [B, 64, 64, 1]
G = gate * generator         [B, 64, 64, d_z]
center c                    [B, d_z]
directions U                [K, d_z]
projection G·U              [B, 64, 64, K]
width = sum(abs(G·U))        [B, K]
center projection c·U        [B, K]
h_plus                      [B, K]
h_minus                     [B, K]
violations                  [B, 2K]
residual r                  [B, 1]
logits                      [B, 1]
```

Core PyTorch sketch:

```python
import math
import torch
import torch.nn as nn
import torch.nn.functional as F


class SupportPolarZonotopeClassifier(nn.Module):
    def __init__(
        self,
        in_channels: int,
        d_token: int = 64,
        d_zono: int = 32,
        n_dirs: int = 32,
        rel_dim: int = 16,
    ):
        super().__init__()

        self.stem = nn.Sequential(
            nn.Conv2d(in_channels, d_token, 3, padding=1),
            nn.GELU(),
            nn.Conv2d(d_token, d_token, 3, padding=1),
            nn.GELU(),
        )

        self.rel = nn.Parameter(torch.randn(64, 64, rel_dim) * 0.02)

        pair_dim = 2 * d_token + rel_dim
        self.gen = nn.Sequential(
            nn.Linear(pair_dim, 128),
            nn.GELU(),
            nn.Linear(128, d_zono),
        )
        self.gate = nn.Sequential(
            nn.Linear(pair_dim, 64),
            nn.GELU(),
            nn.Linear(64, 1),
        )
        self.center = nn.Sequential(
            nn.Linear(d_token, d_zono),
            nn.GELU(),
            nn.Linear(d_zono, d_zono),
        )

        self.raw_dirs = nn.Parameter(torch.randn(n_dirs, d_zono) * 0.05)
        self.raw_beta = nn.Parameter(torch.zeros(n_dirs))
        self.raw_scale = nn.Parameter(torch.tensor(1.0))
        self.bias = nn.Parameter(torch.tensor(0.0))

        mask = torch.ones(64, 64, 1)
        mask[torch.arange(64), torch.arange(64), :] = 0
        self.register_buffer("pair_mask", mask)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        B = x.shape[0]

        h = self.stem(x).flatten(2).transpose(1, 2)  # [B, 64, d_token]

        hi = h[:, :, None, :].expand(B, 64, 64, -1)
        hj = h[:, None, :, :].expand(B, 64, 64, -1)
        rel = self.rel[None, :, :, :].expand(B, 64, 64, -1)

        pair = torch.cat([hi, hj, rel], dim=-1)

        gate = torch.sigmoid(self.gate(pair)) * self.pair_mask[None]
        G = gate * self.gen(pair) / math.sqrt(64 * 63)

        c = self.center(h).mean(dim=1)  # [B, d_zono]

        U = F.normalize(self.raw_dirs, dim=-1)       # [K, d_zono]
        beta = F.softplus(self.raw_beta) + 0.05      # [K]

        proj = torch.einsum("bijd,kd->bijk", G, U)   # [B, 64, 64, K]
        width = proj.abs().sum(dim=(1, 2))           # [B, K]
        cproj = c @ U.T                              # [B, K]

        h_plus = cproj + width
        h_minus = -cproj + width

        v = torch.cat([h_plus - beta, h_minus - beta], dim=-1)
        residual = v.amax(dim=-1, keepdim=True)

        scale = F.softplus(self.raw_scale)
        return scale * residual + self.bias
```

```python
@torch.no_grad()
def extract_certificate(model, x):
    details = model.forward_with_details(x)
    # details["proj"]: [B, 64, 64, K]
    # details["violations"]: [B, 2K]
    win = details["violations"].argmax(dim=-1)
    k = win % details["n_dirs"]
    sign = torch.where(win < details["n_dirs"], 1, -1)

    # For each sample b, score pair (i,j) by abs(proj[b,i,j,k[b]]).
    # Return top pairs and values.
    return {
        "direction_index": k,
        "sign": sign,
        "violation": details["violations"].amax(dim=-1),
    }
```

## 11. Required Ablations

Run all ablations with the same splits, seeds, optimizer, batch size, and training schedule.

1. **No zonotope width:** use only \(c_x\cdot u_k\), removing \(\sum |u_k^\top g_{ij}|\).
2. **Single-square generators:** replace pair generators \(g_{ij}\) with square generators \(g_i\).
3. **Random frozen directions:** initialize \(u_k\), freeze them, and still learn \(\beta_k\).
4. **No polar thresholds:** replace learned \(\beta_k\) with a shared scalar.
5. **One-sided body:** use only \(h_{Z_x}(u_k)\), not \(h_{Z_x}(-u_k)\).
6. **Generic token baseline:** use the same stem and pair MLP parameter budget, but replace support-function residual with learned mean and max token features.
7. **Direction count sweep:** \(K\in\{8,16,32,64\}\).
8. **Generator dimension sweep:** \(d_z\in\{16,32,64\}\).
9. **No relative-square encoding:** remove \(\rho_{ij}\).
10. **Certificate sanity check:** shuffle square-pair contributions at evaluation. The certificate should become less stable and less interpretable.

## 12. Falsification Criteria

Declare the idea unsuccessful if any of these hold:

1. SPZC-Net does not beat the strongest current-board-only baseline by at least **1.5 AUROC points** or **2.0 PR-AUC points** averaged across three seeds.
2. The generic token baseline matches SPZC-Net within **0.5 AUROC points**, suggesting the convex object adds no measurable value.
3. Fine-label diagnostic shows label `2` recall is not improved at matched false-positive rate.
4. Learned \(\beta_k\) values collapse to nearly identical thresholds and learned directions have high cosine similarity, indicating unused polar geometry.
5. The top certificate directions vary wildly under small legal board-preserving augmentations, indicating unstable explanations.
6. Removing zonotope width has no measurable effect, falsifying the sparse-extreme-interaction thesis.
7. Calibration worsens materially relative to the baseline, with no compensating gain in AUROC or PR-AUC.
8. The model's apparent gain disappears when all forbidden fields are removed from the data loader and run artifacts.
