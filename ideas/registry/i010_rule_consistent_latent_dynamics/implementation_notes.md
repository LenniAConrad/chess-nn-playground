# Implementation Notes

## Implementation Plan

1. Implement legal and invalid move descriptor generation.
2. Implement next-board generation for sampled legal moves.
3. Implement shared board encoder and latent transition head.
4. Add auxiliary losses behind config weights.
5. Register `rule_consistent_latent_dynamics`.

## Move Sampling

Use deterministic sampling:

- all checks/captures/promotions first
- then high-value target moves
- then remaining legal moves up to cap

Invalid negatives can be generated by:

- illegal from/to pairs
- wrong piece movement
- own-piece captures
- king-illegal moves

Do not use engine move quality.

## Known Risks

- Next-board generation may slow training.
- Auxiliary losses may dominate.
- Invalid move sampling may be too easy.
- Inference-time sampled move summaries may complicate deployment.

## Testing Plan

- Unit test legal/invalid move descriptor shapes.
- Unit test next-board generation on known FENs.
- Unit test latent transition output shape.
- Unit test auxiliary losses can be disabled.
- Tiny smoke training run.
