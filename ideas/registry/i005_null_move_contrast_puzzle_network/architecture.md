# Architecture

## Architecture Description

Use a shared encoder on two or three deterministic views:

1. current position
2. null-move / side-to-move swapped position
3. optional defensive-turn canonical view

The final classifier uses evidence difference and latent interaction features.

## Input Format

```text
current: batch x 18 x 8 x 8
null_view: batch x 18 x 8 x 8
optional_defensive_view: batch x 18 x 8 x 8
```

The null view must be generated by a single documented function.

## Forward Pass

```text
z_cur = shared_encoder(current)
z_null = shared_encoder(null_view)
e_cur = evidence_head(z_cur)
e_null = evidence_head(z_null)
delta = e_cur - e_null
z_pair = pair_mixer(z_cur, z_null, z_cur - z_null, z_cur * z_null)
logit = classifier([e_cur, e_null, delta, z_pair])
```

## Tensor Shapes

Suggested first version:

```text
encoder_hidden: 96
latent_dim: 128
views: 2
output: batch x 1
```

## Output Heads

Primary:

```text
puzzle_logit
```

Diagnostics:

```text
current_evidence
null_evidence
tempo_contrast_delta
```

## Parameter Estimate

With a shared encoder:

```text
0.8M to 2M parameters
```

Compute is roughly 2x encoder cost.

## FLOP Estimate

```text
O(num_views * encoder_cost + pair_mixer_cost)
```

The first version should be cheaper than response-minimax and obligation-flow models.
